Andrej Karpathy가 "LLM Knowledge Base"라는 개념을 공유했습니다. 쉽게 말하면 이렇습니다.

논문, 기사, 메모, 스크린샷 등을 폴더에 던져넣으면 AI가 알아서 정리된 위키를 만들어주는 시스템입니다. 노션이나 에버노트처럼 사람이 정리하는 게 아닙니다. AI가 전부 합니다.

Karpathy가 설명한 핵심 흐름입니다.

✅ 수집
→ 자료를 한 폴더에 모은다 (정리 안 해도 됨)
→ AI가 주제별로 문서를 자동 생성하고, 문서끼리 연결까지 해줌

✅ 질문
→ 자료가 쌓이면(100개 문서, 40만 단어 수준) 복잡한 질문도 가능
→ "A 자료와 B 자료가 서로 다르게 말하는 부분은?"같은 질문에 답을 찾아줌

✅ 출력
→ 답변을 문서, 슬라이드, 차트 등으로 만들어줌
→ 이 결과물을 다시 위키에 넣으면 지식이 계속 쌓임

✅ 자가 점검
→ AI가 위키 전체를 돌면서 틀린 내용, 빠진 내용, 새로 조사할 주제를 찾아줌

✅ 핵심 원칙
→ "위키를 직접 편집하지 마라. 위키는 AI의 영역이다."

이 개념을 트위터이서 Nick Spisak이 누구나 따라할 수 있게 풀어냈습니다.

필요한 건 폴더 3개와 설명서 파일 1개뿐입니다.

my-knowledge-base/
  raw/       ← 원본 자료를 넣는 곳
  wiki/      ← AI가 정리해서 쓰는 곳
  outputs/   ← AI가 만든 답변·리포트

실행 방법도 간단합니다.

1단계: raw/ 폴더에 자료를 그냥 넣습니다
→ 파일명 정리, 분류 같은 건 안 해도 됩니다

2단계: 설명서 파일(CLAUDE.md)을 만듭니다
→ "이 자료는 OO 주제에 관한 것이다"
→ "문서별로 요약을 넣고 서로 연결해라"
→ AI에게 주는 작업 지시서 역할입니다

3단계: AI에게 "raw/ 전부 읽고 위키 만들어"라고 시킵니다
→ 주제별 문서 생성, 목차 생성, 문서 간 연결까지 자동

4단계: 질문하고, 답변을 다시 위키에 넣습니다
→ 쓸수록 지식이 복리로 쌓이는 구조

Spisak이 추가한 실전 팁도 있습니다.

→ 웹 기사 스크래핑을 자동화하는 도구(agent-browser) 활용법
→ 월 1회 AI 자가 점검으로 틀린 정보가 쌓이는 것을 방지
→ 화려한 도구보다 단순한 폴더 구조가 더 잘 작동합니다

정리하면 이런 흐름입니다.

자료 수집 → AI가 위키로 정리 → 질문 → 답변이 다시 위키로 → 반복

메모 앱을 바꾸는 게 아닙니다. 지식을 관리하는 방식 자체가 바뀌는 겁니다. 사람은 자료를 모으고 질문만 하면 됩니다. 나머지는 AI가 합니다.
