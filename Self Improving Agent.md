Self-improving AI 에이전트가 GitHub 트렌딩에 올라왔길래, 어떻게 동작하는지 살펴봤습니다.

NousResearch의 Hermes Agent라는 오픈소스인데요.
에이전트가 작업을 하면서 경험을 축적하고, 그 경험으로 스스로 나아지는 구조예요.
먼저 전체 구성부터 보고, 핵심인 self-improving이 어떻게 돌아가는지 정리해보겠습니다.

[시스템 구성]
1. 도구 구성
- 사용자 요청을 받으면 tool calling 루프를 돌면서 작업을 수행합니다.
- 웹 검색, 파일 읽기/쓰기, 터미널 실행, 브라우저 자동화 등 50개 넘는 도구를 활용할 수 있습니다.

2. 메모리
- MEMORY.md에 에이전트가 관찰한 사실을, USER.md에 유저 선호도를 저장합니다.
- 세션 기록은 SQLite DB에 남고, 전문 검색으로 과거 대화를 찾아볼 수 있어요.
- 다음 세션이 시작되면 메모리 스냅샷이 시스템 프롬프트에 주입돼서 이전 맥락을 이어갑니다.

3. 인터페이스
- CLI뿐 아니라 Telegram, Discord, Slack, WhatsApp, Signal 등 10개 넘는 플랫폼에서 같은 에이전트를 쓸 수 있어요.
- 하나의 게이트웨이 프로세스가 모든 플랫폼을 처리합니다.

[Self-improving]
Hermes Agent의 Self-improving 전략은 3가지로 구성되어 있습니다.
1. 스킬 시스템
- 시스템 프롬프트에 "복잡한 작업(5+ tool calls)을 끝내면 스킬로 저장하라"는 지침이 들어있어요. LLM이 판단해서 유저 확인 후 스킬 파일로 저장합니다.
- 다음에 비슷한 작업이 들어오면 스킬 목록에서 찾아서 재사용하고, 부정확하면 즉시 패치해요.

2. 진화적 최적화 (GEPA)
- 별도 레포(hermes-agent-self-evolution)에서 DSPy + GEPA(Genetic-Pareto Prompt Evolution)를 활용합니다.
- 실행 실패 기록을 분석해서 원인을 찾고, 스킬이나 프롬프트의 개선된 버전을 자동으로 생성해요. 사람이 리뷰한 뒤 적용 여부를 결정합니다.

3. RL 파이프라인
- 연구용 데이터셋을 에이전트에 배치로 실행해서 trajectory를 생성하고, 이를 RL 학습 데이터로 활용하는 파이프라인이 내장되어 있어요.
- Nous Research의 연구 인프라로 만들어졌지만, 외부 학습 API(Tinker)를 연동하면 누구나 사용할 수 있는 구조입니다.

경험을 데이터화하는 구조는 인상적이었는데,
결국 그 데이터가 좋은 데이터인지 판단하는 기준이 관건일 것 같습니다.
